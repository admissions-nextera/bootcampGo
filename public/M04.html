§1 if / else — Making Decisions

Go's if statement looks almost identical to other languages — with two notable differences that trip up newcomers every time.

❌ Other languages (JS, Java, C)
if (score > 50) {
  // parentheses required
}
✅ Go — no parentheses needed
if score > 50 {
  // clean, no parens
}

The two rules to remember:

1. No parentheses around the condition — Go's formatter will remove them anyway.
2. Braces are mandatory — even for single-line bodies. No optional curly braces in Go.

Let's see a full example:

Goif / else if / else
package main

import "fmt"

func main() {
    score := 72

    if score >= 90 {
        fmt.Println("Grade: A — Outstanding Gopher! 🏆")
    } else if score >= 75 {
        fmt.Println("Grade: B — Solid work. 💪")
    } else if score >= 60 {
        fmt.Println("Grade: C — You passed. Barely. 😬")
    } else {
        fmt.Println("Grade: F — The Forge demands more. 🔥")
    }
}
// Output: Grade: C — You passed. Barely. 😬

The if Initializer — Go's Secret Weapon

Go lets you run a short statement inside the if before the condition. This is called the if initializer, and it's one of the most idiomatic patterns in Go. It keeps your scope clean — the variable only exists inside the if block.

Goif initializer syntax
// Format: if statement; condition { }

if score := getScore(); score >= 75 {
    fmt.Printf("Passed with score: %d\n", score)
} else {
    fmt.Printf("Failed with score: %d\n", score)
}
// score is NOT accessible here — it only lived inside the if block
💡
You'll use this pattern constantly with error handling

In Module 5, you'll learn that Go functions often return a value and an error. The if initializer is the idiomatic way to handle this: if val, err := doSomething(); err != nil { ... }. The variable err stays scoped to the block — no pollution of the outer scope.

§2 for — The Only Loop You'll Ever Need

Here's something that surprises every newcomer: Go has only one loop keyword — for. No while. No do...while. No foreach. Just for, wearing four different costumes depending on how you write it.

😂
Why only one loop?

Because Go's designers looked at every loop pattern in existence and realized they could all be expressed with for. Less to memorize, more to get done. It's the Swiss Army knife of loops — one tool, infinite configurations.

Variant 1 — Classic C-style // aka the "normal for loop" 
// for init; condition; post { }
for i := 0; i < 5; i++ {
    fmt.Printf("Lap %d 🏃\n", i)
}
// Output: Lap 0, Lap 1, Lap 2, Lap 3, Lap 4
Variant 2 — While-style // just the condition, no init or post 
// Go's version of "while (condition)"
lives := 3
for lives > 0 {
    fmt.Printf("Lives remaining: %d\n", lives)
    lives--
}
// Prints: 3, 2, 1 then stops
Variant 3 — Infinite Loop // no condition at all — runs forever until break 
// Useful for servers, games, CLI prompts
attempts := 0
for {
    attempts++
    if attempts >= 3 {
        fmt.Println("Max attempts reached. Bailing out.")
        break  // exit the loop
    }
    fmt.Printf("Attempt %d...\n", attempts)
}
Variant 4 — Range Loop // the Go equivalent of "foreach" 
// Iterate over slices, maps, strings, channels
languages := []string{"Go", "Python", "Rust", "TypeScript"}

for index, value := range languages {
    fmt.Printf("%d: %s\n", index, value)
}
// 0: Go    1: Python    2: Rust    3: TypeScript

// Don't need the index? Use _ to discard it
for _, lang := range languages {
    fmt.Println(lang)
}

break and continue

break exits the loop immediately. continue skips the rest of the current iteration and moves to the next one. Both work the same as in other languages:

Gobreak and continue
for i := 0; i < 10; i++ {
    if i == 3 {
        continue  // skip 3
    }
    if i == 7 {
        break     // stop at 7
    }
    fmt.Print(i, " ")
}
// Output: 0 1 2 4 5 6

§3 switch — Cleaner than a Chain of if/else

Go's switch is like a smarter, cleaner version of what you've seen in other languages. The big difference: Go's cases don't fall through by default. No more forgetting a break statement and accidentally running the next case.

❌ Other languages (C, Java, JS)

Cases fall through automatically. You must add break to every case or risk running into the next one accidentally. A famous source of bugs.

✅ Go

Cases stop automatically. If you want fallthrough, you explicitly write fallthrough. Safe by default, intentional when needed.

Gobasic switch
package main

import "fmt"

func main() {
    day := "Friday"

    switch day {
    case "Monday":
        fmt.Println("Start of the grind. ☕")
    case "Wednesday":
        fmt.Println("Hump day. You're halfway there.")
    case "Friday":
        fmt.Println("It's Friday! Ship it. 🚀")
    case "Saturday", "Sunday":  // multiple values in one case
        fmt.Println("Weekend Bootcamp mode. 🔨")
    default:
        fmt.Println("Just another day.")
    }
}
// Output: It's Friday! Ship it. 🚀

Switch Without a Condition — The Cleaner if/else Chain

One of Go's most elegant patterns: a switch with no expression at all. Each case becomes a condition. This is often cleaner than a long if / else if / else if chain:

Go expressionless switch — annotated
score := 72
Our value to evaluate. Could come from user input, a calculation, anything.
switch { // no expression here
No value after switch. Each case will provide its own boolean condition.
case score >= 90:
Checked first. If true, runs its body and exits the switch.
fmt.Println("A grade")
Only runs if the case above is true.
case score >= 75:
Only reached if the first case was false.
fmt.Println("B grade")
case score >= 60:
fmt.Println("C grade")
This one runs. score(72) is ≥ 60, and no earlier case matched.
default:
Catches everything that didn't match any case. Like else in an if chain.
fmt.Println("F grade")
}
End of switch. Execution continues after this point.

§4 defer — Do This Later, I Promise

defer is one of Go's most unique and genuinely useful features. It schedules a function call to run just before the surrounding function returns — no matter how that function exits. Whether it returns normally, hits an error, or panics, deferred calls always run.

Gobasic defer
func main() {
    fmt.Println("1. Starting up...")

    defer fmt.Println("3. Deferred: this runs last")

    fmt.Println("2. Doing work...")
}

// Output:
// 1. Starting up...
// 2. Doing work...
// 3. Deferred: this runs last

Multiple defers stack up in LIFO order (Last In, First Out) — like a pile of plates. The last defer you wrote runs first:

Gomultiple defers — LIFO order
func main() {
    defer fmt.Println("deferred 1 — runs LAST")
    defer fmt.Println("deferred 2 — runs second")
    defer fmt.Println("deferred 3 — runs FIRST")

    fmt.Println("main body")
}
// Output:
// main body
// deferred 3 — runs FIRST
// deferred 2 — runs second
// deferred 1 — runs LAST

The Real Use Case: Cleanup

Defer's killer use case is cleanup. When you open a file, a network connection, or a database handle — you want to guarantee it gets closed when you're done, no matter what. Without defer, you'd have to remember to close it before every possible return path. With defer, you close it right after you open it — and Go handles the rest:

Godefer for cleanup — the real pattern
func processFile(filename string) {
    file, err := os.Open(filename)
    if err != nil {
        fmt.Println("Error opening file:", err)
        return
    }
    defer file.Close()  // ← scheduled right after Open. Will run no matter what.

    // Now do work with the file...
    // Even if this code panics or returns early,
    // file.Close() is GUARANTEED to run.
    fmt.Println("Processing file...")
}
💡
The Open → defer Close pattern is sacred Go idiom

Any time you see resource, err := open/connect/create() in Go, the very next line is almost always defer resource.Close(). You'll write this pattern hundreds of times in your Go career. Start now, get comfortable.

§5 Putting It All Together

Let's build a small program that uses if, for, switch, and defer together — a simple number guessing game engine. Type it out and run it:

Gofizzbuzz_plus.go — a classic, Go-style
package main

import "fmt"

func main() {
    defer fmt.Println("\n✅ FizzBuzz complete!")  // runs last, guaranteed

    fmt.Println("FizzBuzz from 1 to 20:\n")

    for i := 1; i <= 20; i++ {
        switch {
        case i%15 == 0:
            fmt.Printf("%2d → FizzBuzz 🎉\n", i)
        case i%3 == 0:
            fmt.Printf("%2d → Fizz\n", i)
        case i%5 == 0:
            fmt.Printf("%2d → Buzz\n", i)
        default:
            fmt.Printf("%2d\n", i)
        }
    }
}
