§1 Arrays — Fixed, Rigid, Rarely Used Directly

An array in Go is a fixed-size, ordered collection of values of the same type. The size is part of the type — a [3]int and a [5]int are completely different types. You cannot resize an array after it's created.

Goarrays — declaration and access
// Declare an array of 3 strings
var languages [3]string
languages[0] = "Go"
languages[1] = "Python"
languages[2] = "Rust"

fmt.Println(languages[0])   // Go
fmt.Println(len(languages))  // 3

// Short declaration with values inline
scores := [4]int{88, 92, 76, 95}

// Let Go count the elements with [...]
names := [...]string{"Anas", "Sarah", "Omar"}
fmt.Println(len(names))  // 3

// Iterate with range
for i, s := range scores {
    fmt.Printf("scores[%d] = %d\n", i, s)
}

⚠️
Arrays are value types — they copy on assignment

When you assign an array to another variable or pass it to a function, Go makes a complete copy. Changing the copy doesn't affect the original. This is different from most languages — and it's exactly why slices exist.

🤔
So when do I actually use arrays?

Rarely directly. Arrays are the low-level building block that slices are built on top of. In practice, you'll almost always reach for a slice. But understanding arrays makes slices click instantly — so pay attention.

§2 Slices — The Star of the Show 🌟

A slice is a dynamically-sized, flexible view into an array. It's the data structure you'll use in 95% of your Go code whenever you need a list of things. Unlike arrays, slices can grow, shrink, and be passed around without copying all the data.

A slice has three components under the hood: a pointer to the underlying array, a length (how many elements it currently holds), and a capacity (how many elements the underlying array can hold before Go needs to allocate a bigger one).

Creating Slices

Gothree ways to create a slice
// 1. Slice literal — most common
names := []string{"Anas", "Sarah", "Omar"}

// 2. make() — create with length and optional capacity
// make([]Type, length, capacity)
scores := make([]int, 5)        // len=5, cap=5, all zeros
buffer := make([]byte, 0, 100)   // len=0, cap=100 — pre-allocated

// 3. nil slice — zero value, valid and usable
var items []string  // nil, len=0, cap=0
fmt.Println(items == nil)  // true
fmt.Println(len(items))    // 0 — safe to use immediately

The Essential Slice Operations

Goappend, len, cap, and slicing
fruits := []string{"apple", "banana", "cherry"}

// append — adds elements, returns a NEW slice
fruits = append(fruits, "date")
fruits = append(fruits, "elderberry", "fig")  // append multiple at once
fmt.Println(fruits)   // [apple banana cherry date elderberry fig]
fmt.Println(len(fruits))  // 6

// Slicing — create a sub-slice with [low:high]
// low is inclusive, high is exclusive
first3 := fruits[0:3]   // [apple banana cherry]
last2  := fruits[4:]    // [elderberry fig]  (high defaults to len)
middle := fruits[1:4]   // [banana cherry date]
all    := fruits[:]      // all elements (copy of slice header)

fmt.Println(first3, last2, middle)
⚠️
Slices share the underlying array — this will bite you once

When you slice a slice (b := a[1:3]), both a and b point to the same underlying array. Modifying b[0] will also change a[1]. To get a true independent copy, use the built-in copy() function.

Gocopy() — making an independent slice
original := []int{1, 2, 3, 4, 5}

// This shares the array — dangerous
shared := original[1:4]
shared[0] = 99
fmt.Println(original)  // [1 99 3 4 5] — original was modified!

// This makes a true independent copy — safe
clone := make([]int, len(original))
copy(clone, original)
clone[0] = 777
fmt.Println(original)  // [1 99 3 4 5] — original unchanged
fmt.Println(clone)     // [777 99 3 4 5]

Slice Cheat Sheet

OperationSyntaxWhat it does

length len(s) Number of elements currently in the slice.
capacity cap(s) Size of the underlying array. Grows automatically.
sub-slice s[low:high] Elements from index low (inclusive) to high (exclusive).
copy copy(dst, src) Deep copy into dst. Returns number of elements copied.

Go deleting an element from a slice — annotated
s := []string{"a", "b", "c", "d", "e"}
Our slice: indices 0–4.
i := 2 // delete index 2 ("c")
We want to remove "c" at position 2.

s[:2] = ["a","b"]. s[3:] = ["d","e"]. Spread and append together. Result: ["a","b","d","e"].

§3 Maps — Key-Value Storage 🗺️

A map stores key-value pairs. Give it a key, get back a value instantly. It's Go's equivalent of a Python dictionary, JavaScript object, or Java HashMap. Keys can be any comparable type (strings, ints, etc.), and values can be anything.

Gocreating and using maps
// map[KeyType]ValueType

// Map literal — initialize with values
scores := map[string]int{
    "Anas":  95,
    "Sarah": 88,
    "Omar":  72,
}

// Access a value by key
fmt.Println(scores["Anas"])   // 95

// Add or update a key
scores["Layla"] = 91           // add new
scores["Omar"]  = 85           // update existing

// Delete a key
delete(scores, "Sarah")

// Iterate over a map (order is random — by design)
for name, score := range scores {
    fmt.Printf("%s: %d\n", name, score)
}

The Safe Lookup — The Comma-OK Pattern

Here's the critical thing about maps: if you access a key that doesn't exist, Go returns the zero value for the value type — silently. This means accessing a missing key doesn't crash — but it can give you unexpected results. The solution is Go's comma-ok idiom:

Gothe comma-ok pattern — always use this for map lookups
scores := map[string]int{"Anas": 95}

// ❌ Dangerous — returns 0 if key missing, silently
score := scores["Ghost"]
fmt.Println(score)  // 0 — but is Ghost a real student with score 0?

// ✅ Safe — the comma-ok pattern
score, ok := scores["Anas"]
if ok {
    fmt.Printf("Anas scored %d\n", score)
} else {
    fmt.Println("Student not found")
}

// Short form — just checking existence
if _, ok := scores["Ghost"]; !ok {
    fmt.Println("Ghost is not a real student (probably)")
}
💡
The comma-ok pattern is everywhere in Go

You'll see this same two-value lookup pattern in type assertions, channel receives, and map lookups. Once you learn it once, you recognize it everywhere. It's one of Go's most consistent idioms.

Maps of Slices — A Powerful Combo

One of the most useful patterns in Go is a map where the values are slices. This lets you group multiple items under the same key — like a dictionary of lists:

Gomap of slices — grouping data
// Group students by their grade
gradeBook := map[string][]string{}

gradeBook["A"] = append(gradeBook["A"], "Anas")
gradeBook["A"] = append(gradeBook["A"], "Layla")
gradeBook["B"] = append(gradeBook["B"], "Omar")
gradeBook["C"] = append(gradeBook["C"], "Kareem")

for grade, students := range gradeBook {
    fmt.Printf("Grade %s: %v\n", grade, students)
}
// Grade A: [Anas Layla]
// Grade B: [Omar]
// Grade C: [Kareem]
👀
Maps are reference types

Unlike arrays, maps are reference types. When you pass a map to a function and modify it, the original is modified too. There's no hidden copy — you're always working with the same underlying data. This is usually what you want, but it's good to know.

§4 Putting It All Together

Let's write a program that uses arrays, slices, and maps together — a simple student grade tracker that stores scores, calculates averages, and groups students by performance.

Gograde_tracker.go — type this and run it
package main

import "fmt"

func average(scores []int) float64 {
    if len(scores) == 0 {
        return 0
    }
    total := 0
    for _, s := range scores {
        total += s
    }
    return float64(total) / float64(len(scores))
}

func rank(avg float64) string {
    switch {
    case avg >= 90: return "🏆 Forge Master"
    case avg >= 75: return "⚔️  Senior Gopher"
    case avg >= 60: return "🔨 Apprentice"
    default:        return "📚 Keep Studying"
    }
}

func main() {
    // Map of student name → slice of their scores
    students := map[string][]int{
        "Anas":   {95, 88, 92, 97},
        "Sarah":  {78, 82, 75, 80},
        "Omar":   {62, 58, 70, 65},
        "Layla":  {91, 96, 89, 94},
    }

    fmt.Println("=== Gopher's Forge — Grade Report ===")
    fmt.Println()

    // Print each student's average and rank
    for name, scores := range students {
        avg := average(scores)
        fmt.Printf("%-8s avg: %5.1f  %s\n", name, avg, rank(avg))
    }

    fmt.Println()

    // Add a new score for Anas mid-bootcamp
    students["Anas"] = append(students["Anas"], 100)
    newAvg := average(students["Anas"])
    fmt.Printf("Anas just aced M06! New avg: %.1f — %s\n", newAvg, rank(newAvg))
}

§5 Arrays vs Slices vs Maps — Quick Reference

📦 Arrays

  • Fixed size, set at compile time
  • Size is part of the type
  • Value type — copies on assign
  • Rarely used directly
  • Use when size is truly fixed

🌟 Slices

  • Dynamic size, grows with append
  • Built on top of arrays
  • Reference type — shares data
  • Your go-to list type
  • Use for almost everything

🗺️ Maps

  • Key-value pairs
  • Keys must be comparable types
  • Reference type — no hidden copy
  • Iteration order is random
  • Use the comma-ok for lookups
