Core language fundamentals

§1 What Is a Variable, Really?

A variable is a named box that holds a value. That's it. You give the box a name, put something inside it, and refer to it by name later. Go just happens to be very particular about what kind of thing goes in each box.

In Go, every variable has a type, and that type is locked in at compile time. You can't put a number in a box that was built for text. Go won't allow it — and it'll tell you about it loudly before your program ever runs.

😄
Why does this matter?

In JavaScript, "5" + 3 gives you "53". In Python it throws an error at runtime. In Go it refuses to compile entirely. One of these will save you at 3am when you're debugging production. Spoiler: it's Go.

§2 Three Ways to Declare a Variable

Go gives you three ways to create a variable. They're not interchangeable — each has its place. Let's meet all three.

Way 1 — The Full Declaration (var)

This is the most explicit form. You're telling Go: "I want a variable, here's its name, here's its type, and here's its value."

Govar declaration
var name string = "Anas"
var age  int    = 25
var isForging bool = true

fmt.Println(name, age, isForging)
// Output: Anas 25 true

You can also declare multiple variables at once with a var block — a pattern you'll often see at the top of a file:

Govar block
var (
    name      string = "Anas"
    age       int    = 25
    language  string = "Go"
    isForging bool   = true
)

Way 2 — Type Inference (var without type)

Go is smart enough to figure out the type from the value you assign. You can drop the type annotation and Go will infer it:

Gotype inference with var
var name = "Anas"      // Go infers: string
var age  = 25          // Go infers: int
var pi   = 3.14159    // Go infers: float64

// Want to verify the type? fmt.Printf can show you:
fmt.Printf("name is type: %T\n", name)  // string
fmt.Printf("age  is type: %T\n", age)   // int
fmt.Printf("pi   is type: %T\n", pi)    // float64
👀
The %T format verb

%T in fmt.Printf prints the type of a value. It's an incredibly useful debugging tool — whenever you're unsure what type Go inferred, just %T it.

Way 3 — The Short Declaration (:=) ⭐ Most Common

This is the one you'll use 90% of the time inside functions. The := operator declares and assigns in one shot, with full type inference. No var, no type — just a name and a value:

Goshort declaration — the Go way
name := "Anas"       // string, inferred
age  := 25           // int, inferred
pi   := 3.14159     // float64, inferred

fmt.Println(name, age, pi)
// Output: Anas 25 3.14159
⚠️
:= only works inside functions

The short declaration := cannot be used at the package level (outside of any function). For package-level variables, you must use var. If you try := at the top of a file, Go will refuse to compile.

Here's a quick side-by-side so you can see all three styles together:

❌ More verbose than needed
var name string = "Anas"
var age int = 25
✅ Idiomatic Go
name := "Anas"
age  := 25

§3 Go's Core Types — Your Toolkit

Go has a focused set of built-in types. Unlike some languages that give you fifteen flavors of integer and let you choose the wrong one, Go's types are practical and well-named. Here are the ones you'll actually use:

string
A sequence of characters. Always UTF-8. Immutable.
"Hello, Gopher!"
int
A whole number. Size depends on your OS (64-bit on 64-bit systems). Your default integer type.
42, -7, 0
int8 / int16 / int32 / int64
Fixed-size integers when you need precise memory control.
var x int64 = 100
float64
A decimal number. This is Go's default floating point type. Use this unless you have a specific reason not to.
3.14, -0.5, 99.9
float32
A smaller decimal number. Less precise than float64. Rarely needed unless optimizing memory.
var f float32 = 3.14
bool
Either true or false. No truthy/falsy nonsense — just those two exact values.
true, false
byte
An alias for uint8. Used when working with raw data or individual characters.
var b byte = 'A'
rune
An alias for int32. Represents a Unicode code point. Used when working with individual characters.
var r rune = '🔨'
💡
The 80/20 Rule for Types

In 80% of Go code, you'll only use: string, int, float64, and bool. The rest exist for specific situations. Start with these four and reach for the others only when you need them.

§4 Zero Values — Go's Safety Net

Here's one of Go's most quietly brilliant features: every variable in Go has a zero value. If you declare a variable without assigning it a value, Go doesn't leave it as some unpredictable garbage value from memory (looking at you, C). It initializes it to a sensible default automatically.

Gozero values in action
var name    string
var age     int
var balance float64
var active  bool

fmt.Printf("name:    %q\n", name)     // ""
fmt.Printf("age:     %d\n", age)      // 0
fmt.Printf("balance: %f\n", balance)  // 0.000000
fmt.Printf("active:  %v\n", active)   // false
Type Zero Value What it means
string "" An empty string — not null, not undefined. Empty.
int, int64, etc. 0 The number zero.
float64 0.0 The decimal zero.
bool false Boolean false. Not "null" or "undefined" — just false.
pointer / slice / map nil The explicit absence of a value. We'll cover this later.
😂
No More "undefined is not a function"

JavaScript developers: that nauseating moment when something is undefined and you can't figure out why? That doesn't happen in Go. Everything has a value. Always. Your stress levels at 2am will thank you.

§5 Type Safety — Go Doesn't Do Implicit Conversion

Go is a statically typed language — which means once a variable has a type, it stays that type forever. And Go will never silently convert between types. You have to do it explicitly, or it won't compile.

Let's see what this looks like when you try to mix types:

Gotype mismatch — this WILL NOT compile
age    := 25           // int
weight := 72.5         // float64

// ❌ This will fail — can't add int and float64
total := age + weight
terminalcompiler error
./main.go:7:16: invalid operation: age + weight
(mismatched types int and float64)

To fix this, you explicitly convert one type to match the other. In Go, type conversion looks like a function call — TypeName(value):

Goexplicit type conversion — the correct way
age    := 25
weight := 72.5

// ✅ Convert age (int) to float64 first, then add
total := float64(age) + weight

fmt.Printf("Total: %.1f\n", total)
// Output: Total: 97.5
Go type conversion — annotated
age := 25
Go infers type int from the value 25.
weight := 72.5
Go infers type float64 from 72.5.
float64(age)
Explicit conversion: creates a new float64 value from age. The original age variable is still an int.
total := float64(age) + weight
Now both sides are float64 — Go is happy. total is inferred as float64.
fmt.Printf("%.1f\n", total)
The %.1f format verb prints a float with 1 decimal place. So 97.5, not 97.500000.

§6 Constants — Values That Never Change

A constant is a value that's set once and can never be reassigned. You declare it with const instead of var, and Go will refuse to compile if you ever try to change it.

Goconstants
const Pi         = 3.14159
const MaxRetries = 3
const AppName    = "Gopher's Forge"
const Version    = "1.0.0"

// You can also group constants, just like var
const (
    StatusOK       = 200
    StatusNotFound = 404
    StatusError    = 500
)

fmt.Println(AppName, Version)
// Output: Gopher's Forge 1.0.0

Try to reassign a constant and the compiler shuts you down immediately:

Gothis will NOT compile
const MaxRetries = 3
MaxRetries = 5  // ❌ cannot assign to MaxRetries (untyped int constant)

iota — The Constant Superpower

Go has a special keyword called iota that auto-increments inside a const block. It's perfect for creating sets of related constants like status codes, directions, or day names — without typing the numbers yourself.

Goiota — auto-incrementing constants
const (
    Sunday    = iota  // 0
    Monday              // 1
    Tuesday             // 2
    Wednesday           // 3
    Thursday            // 4
    Friday              // 5
    Saturday            // 6
)

fmt.Println(Monday, Friday)
// Output: 1 5

You can even do math with iota. A classic use case is representing file sizes — multiply iota by a power of 1024:

Goiota with expressions
const (
    _           = iota              // skip 0 with blank identifier
    KB          = 1 << (10 * iota) // 1 << 10 = 1,024
    MB                               // 1 << 20 = 1,048,576
    GB                               // 1 << 30 = 1,073,741,824
)

fmt.Printf("1 KB = %d bytes\n", KB)
fmt.Printf("1 MB = %d bytes\n", MB)
fmt.Printf("1 GB = %d bytes\n", GB)
💡
The blank identifier _

That lone underscore _ is Go's "I deliberately don't care about this value" symbol. It's called the blank identifier. You'll see it everywhere in Go — anytime you want to discard a value without using it. In the example above, we used it to skip iota's initial value of 0.

§7 Reassigning Variables & Multiple Assignment

Once a variable is declared, you reassign it with plain = (no colon). The := is only for the first time — it declares and assigns. After that, just =.

Godeclaring vs reassigning
score := 0          // ← declare with :=
fmt.Println(score)  // 0

score = 100          // ← reassign with =
fmt.Println(score)  // 100

score = score + 50   // ← or use += shorthand below
score += 50
fmt.Println(score)  // 200

Go also supports multiple assignment in a single line — a feature that becomes incredibly useful for functions that return multiple values (which we'll cover in Module 5):

Gomultiple assignment
// Declare and assign multiple variables at once
x, y := 10, 20
fmt.Println(x, y)  // 10 20

// Swap two variables without a temp variable — idiomatic Go
x, y = y, x
fmt.Println(x, y)  // 20 10
🤯
Swapping without a temp variable

In most languages, swapping two variables requires a temporary third variable. In Go, x, y = y, x just works. Go evaluates the right side completely before assigning. Clean, elegant, and satisfying every single time.

§8 Putting It All Together

Let's write a small program that uses everything from this module — variables, types, constants, and multiple assignment — in a realistic context.

Goprofile.go — type this and run it
package main

import "fmt"

const BootcampName = "Gopher's Forge"
const Version      = "1.0"

func main() {
    // Declare using short syntax
    name     := "Anas"
    age      := 25
    language := "Go"
    score    := 0

    // Print initial profile
    fmt.Printf("=== %s v%s ===\n", BootcampName, Version)
    fmt.Printf("Student : %s (age %d)\n", name, age)
    fmt.Printf("Language: %s\n", language)
    fmt.Printf("Score   : %d\n\n", score)

    // Update score after completing a module
    score += 10
    fmt.Printf("✅ Module complete! New score: %d\n", score)

    // Multiple assignment
    x, y := 100, 200
    fmt.Printf("Before swap: x=%d, y=%d\n", x, y)
    x, y = y, x
    fmt.Printf("After swap:  x=%d, y=%d\n", x, y)
}
terminalexpected output
=== Gopher's Forge v1.0 ===
Student : Anas (age 25)
Language: Go
Score   : 0

✅ Module complete! New score: 10
Before swap: x=100, y=200
After swap:  x=200, y=100
