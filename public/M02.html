§1 Installing Go

Go has one of the friendliest installation experiences of any language. No version manager drama, no PATH nightmares (well, almost none), no dependency hell. There's one installer, it works, and you move on with your life.

Head to go.dev/dl and download the installer for your operating system. Then follow the steps below for your platform:

🍎 macOS

1

Download the .pkg installer

From go.dev/dl, grab the latest .pkg file for macOS (Apple Silicon or Intel — pick the right one for your chip).

2

Run the installer

Double-click, click through the prompts. Go installs itself to /usr/local/go. No configuration needed.

3

Verify it worked

Open Terminal and run:

terminalverify installation
$ go version
go version go1.22.0 darwin/arm64

If you see a version number, you're golden. 🎉

🪟 Windows

1

Download the .msi installer

From go.dev/dl, grab the .msi file. Run it, click Next a few times. Go installs to C:\Program Files\Go and adds itself to your PATH automatically.

2

Open PowerShell or CMD and verify

powershellverify installation
> go version
go version go1.22.0 windows/amd64

You'll need to open a new terminal window after installing for PATH changes to take effect.

🐧 Linux

1

Download and extract the tarball

terminallinux installation
# Remove any old Go installation first
$ sudo rm -rf /usr/local/go

# Download and extract (replace with latest version)
$ wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz
$ sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz
2

Add Go to your PATH

Add this line to your ~/.bashrc or ~/.zshrc:

bash~/.bashrc or ~/.zshrc
export PATH=$PATH:/usr/local/go/bin

Then reload: source ~/.bashrc and verify with go version.

⚠️
If go version says "command not found"

This is almost always a PATH issue. Close and reopen your terminal, then try again. If it still fails, check that /usr/local/go/bin (macOS/Linux) or C:\Program Files\Go\bin (Windows) is in your PATH environment variable.

§2 Setting Up Your Editor

Go works with any editor, but VS Code is the recommended choice for this bootcamp. It has the best Go extension in the game and it's free.

1

Install VS Code

If you don't have it, grab it from code.visualstudio.com. Takes two minutes.

2

Install the Go extension

Open VS Code → press Ctrl+Shift+X (or Cmd+Shift+X on Mac) → search for "Go" → install the one by Go Team at Google. It has a little gopher icon. You can't miss it.

3

Install Go tools

When you first open a .go file, VS Code will ask you to install additional Go tools. Click "Install All". This gives you autocompletion, formatting, error highlighting — everything you need.

💡
gopls is your best friend

The Go language server (gopls) powers all the smart features in VS Code. It's what gives you instant error feedback as you type — before you even save the file. Go's tooling is genuinely excellent.

§3 Your First Go Project

Time to create something. In Go, every project lives inside a module. A module is just a folder with a special file called go.mod that tells Go who owns this project and what it depends on. Think of it as your project's ID card.

1

Create a project folder

terminal
$ mkdir hello-gopher
$ cd hello-gopher
2

Initialize a Go module

terminal
$ go mod init hello-gopher
go: creating new go.mod: module hello-gopher

This creates a go.mod file. Open it and you'll see:

go.modauto-generated
module hello-gopher

go 1.22

That's it. Two lines. Your project is initialized. Go doesn't believe in ceremony.

3

Open the folder in VS Code

terminal
$ code .

Create a new file called main.go. This is where we write our first program.

§4 Writing Hello, Gopher — Line by Line

Here's the moment. Type this out — don't copy-paste. Your fingers need to learn the syntax just as much as your brain does. Every character matters, and typing it yourself forces you to actually look at it.

Go main.go — type this out 
package main

import "fmt"

func main() {
    fmt.Println("Hello, Gopher! Welcome to the Forge. 🔨")
}

Now let's go through every single line like a surgeon. Nothing gets skipped.

Go main.go — annotated
package main
Every .go file must belong to a package. main is the special package that makes a runnable program.
Blank lines are just breathing room. Go ignores them.
import "fmt"
fmt is Go's formatting package. It handles printing to the console. We import only what we use — Go won't compile if you import something unused.
Another breath.
func main() {
func declares a function. main is the entry point — Go always starts execution here. The opening { MUST be on the same line. This is enforced by Go's formatter.
fmt.Println("Hello, Gopher!")
fmt.Println prints text followed by a newline. The fmt. is the package name, Println is the function. Indentation is 1 tab (Go's gofmt will enforce this).
}
Closes the main function. Simple, clean, done.
🤓
Why does the opening brace have to be on the same line?

Because Go automatically inserts semicolons at the end of certain lines during compilation. If you put { on a new line, Go would insert a semicolon before it, breaking the syntax. The formatter (gofmt) enforces this so nobody has to argue about it. Ever. Brace wars: solved.

§5 Running Your Program

Go gives you two ways to run your code. You'll use both throughout this bootcamp, so let's learn them now.

Option A: go run — Fast & Temporary

go run compiles and runs your code in one step without creating a binary file. It's perfect for development — quick to iterate, nothing left behind.

terminalfrom inside hello-gopher/
$ go run main.go
Hello, Gopher! Welcome to the Forge. 🔨

That's it. One command. Your first Go program ran. Breathe that in for a second. 🐹

Option B: go build — Real Binary

go build compiles your code into a real standalone binary — an executable file you can run anywhere, even on machines with no Go installed.

terminalbuild a real binary
$ go build -o hello-gopher main.go

# Now run the binary directly
$ ./hello-gopher
Hello, Gopher! Welcome to the Forge. 🔨

# On Windows, run it like this instead:
> hello-gopher.exe
💡
Which one should you use?

During development → go run. When you want to ship something → go build. For this bootcamp, we'll mostly use go run so we can stay in the flow without extra steps.

§6 Meet gofmt — Your Auto-Formatter

One of Go's most underrated superpowers is gofmt — a built-in code formatter that every Go developer in the world uses. There are no arguments about tabs vs spaces, no Prettier config files, no style debates. Everyone runs gofmt and the code looks the same. Always.

VS Code with the Go extension runs gofmt automatically on every save. But let's see what it does manually. Deliberately write ugly code:

Gougly.go — intentionally bad formatting
package main
import "fmt"
func main(){
fmt.Println(   "I wrote this badly on purpose"   )
}

Now run:

terminal
$ gofmt -w ugly.go

Open the file again. It's been cleaned up automatically:

Gougly.go — after gofmt (now beautiful)
package main

import "fmt"

func main() {
    fmt.Println("I wrote this badly on purpose")
}
😂
Anas's Take

gofmt is basically a coworker who silently fixes your indentation every time you save, never complains about it, and never asks for credit. This is the coworker everyone deserves. Cherish it.

§7 Let's Break It On Purpose

You can't truly understand a language until you've seen it scream at you. Let's trigger the two most common beginner errors right now, so they never catch you off guard again.

Error #1 — Importing something you don't use

Gobroken.go
package main

import (
    "fmt"
    "math"  // ← we're importing this but not using it
)

func main() {
    fmt.Println("Hello")
}
terminalwhat Go says
./broken.go:5:2: "math" imported and not used

Go refuses to compile. Not a warning — a hard error. This is a feature, not a bug. Unused imports are dead weight that slow down compilation and confuse readers. Go doesn't negotiate on this.

Error #2 — Declaring a variable you never use

Gobroken2.go
package main

import "fmt"

func main() {
    name := "Anas"  // ← declared but never used
    fmt.Println("Hello, Gopher!")
}
terminalwhat Go says
./broken2.go:6:2: name declared and not used

Same deal. Go will not let you litter your code with unused variables. Fix it by actually using name:

Gofixed.go
package main

import "fmt"

func main() {
    name := "Anas"
    fmt.Printf("Hello, %s! Welcome to the Forge.\n", name)
}
terminal
$ go run fixed.go
Hello, Anas! Welcome to the Forge.
👀
Notice something new? fmt.Printf

We snuck in Printf — it's like Println but with formatting. The %s is a placeholder for a string, and \n is a newline. We'll use Printf a lot — it's how you build dynamic output strings in Go.

§8 Try It Yourself

Before we move on, do this small exercise. Modify your main.go to print a personalized greeting. Use your own name:

Goyour turn — modify main.go
package main

import "fmt"

func main() {
    name := "YOUR NAME HERE"  // ← put your name in
    language := "Go"

    fmt.Printf("Hey %s! You're learning %s this weekend.\n", name, language)
    fmt.Println("Let's build something awesome. 🔨")
}

Run it with go run main.go. It should say:

terminalexpected output
Hey Anas! You're learning Go this weekend.
Let's build something awesome. 🔨

If it does — you just wrote, ran, and customized a Go program. That's not nothing. That's the beginning of everything.
