§1 How HTTP Works in Go — The Core Idea

Go's HTTP model is built on one interface: http.Handler. Anything that implements a single method — ServeHTTP(w http.ResponseWriter, r *http.Request) — can handle HTTP requests. The entire framework ecosystem is just wrappers around this one interface.

HTTP Request (r *http.Request)
r.Method "POST"
r.URL.Path "/api/users"
r.Header Content-Type…
r.Body JSON payload
r.URL.Query() ?page=2
→
HTTP Response (w http.ResponseWriter)
w.Header().Set() set headers
w.WriteHeader() status code
w.Write() body bytes
json.NewEncoder(w) stream JSON
 
👀
ResponseWriter is write-once, top-to-bottom

Headers must be set before WriteHeader(). WriteHeader() must be called before Write(). Once you call Write(), the response has begun and headers are locked. This order is enforced by the HTTP protocol — not Go. Most bugs come from writing the body before setting a header or status code.

§2 Your First HTTP Server

Goa minimal HTTP server — 10 lines
package main

import (
    "fmt"
    "net/http"
)

func helloHandler(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintf(w, "Hello, Gopher! You hit: %s\n", r.URL.Path)
}

func main() {
    http.HandleFunc("/", helloHandler)

    fmt.Println("Server running on http://localhost:8080")
    http.ListenAndServe(":8080", nil)
}
terminalrun it and test it

§3 JSON — The Language of APIs

Go's encoding/json package converts between Go structs and JSON using struct tags — small annotations on struct fields that tell the JSON encoder what key name to use:

Go struct tags and JSON encoding — annotated
type User struct {
ID int `json:"id"`
Struct tag: field ID becomes "id" in JSON. Tags are backtick strings — raw literals.
Name string `json:"name"`
Without tags, Go uses the field name as-is. Tags let you control casing (Go uses PascalCase, JSON uses camelCase).
Email string `json:"email"`
Pass string `json:"-"`
"-" means "never include this field in JSON output." Critical for passwords, secrets, internal fields.
Score int `json:"score,omitempty"`
omitempty: skip this field if it's the zero value (0, "", false, nil). Keeps responses lean.
}
Goencoding and decoding JSON
// Encoding — Go struct → JSON bytes
u := User{ID: 1, Name: "Anas", Email: "anas@forge.go", Pass: "secret"}
data, err := json.Marshal(u)
fmt.Println(string(data))
// {"id":1,"name":"Anas","email":"anas@forge.go"}  ← Pass omitted!

// Decoding — JSON bytes → Go struct
jsonStr := `{"id":2,"name":"Sarah","email":"sarah@forge.go"}`
var u2 User
err = json.Unmarshal([]byte(jsonStr), &u2)
fmt.Println(u2.Name)  // Sarah

// In HTTP handlers — use Encoder/Decoder for streaming (more efficient)
func handler(w http.ResponseWriter, r *http.Request) {
    // Decode request body
    var input User
    json.NewDecoder(r.Body).Decode(&input)

    // Encode response
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(input)
}

§4 Routing — Handling Different Paths & Methods

Go 1.22 upgraded the standard library's ServeMux router to support method matching and path parameters natively — no third-party router needed for most APIs:

GoGo 1.22+ routing with method + path params
func main() {
    mux := http.NewServeMux()

    // Method + path: "METHOD /path"
    mux.HandleFunc("GET /api/users",        listUsers)
    mux.HandleFunc("POST /api/users",       createUser)
    mux.HandleFunc("GET /api/users/{id}",   getUser)    // {id} = path param
    mux.HandleFunc("DELETE /api/users/{id}", deleteUser)

    http.ListenAndServe(":8080", mux)
}

// Access the path parameter in the handler
func getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")    // Go 1.22+
    fmt.Fprintf(w, "Getting user: %s\n", id)
}
💡
Go 1.22 changed the game for stdlib routing

Before 1.22, the standard ServeMux had no method matching or path parameters — developers always reached for chi or gorilla/mux. Since 1.22, for simple APIs you genuinely don't need a framework. We'll use stdlib for the capstone. If you need regex patterns or middleware chains, chi is still a great choice at just ~1,400 lines of code.

§5 A Real CRUD API — Users Service

Let's build a complete in-memory Users CRUD API with 4 endpoints. This is the full pattern you'll use in the capstone:

MethodPathAction
GET/api/usersList all users
POST/api/usersCreate a new user
GET/api/users/{id}Get a single user by ID
DELETE/api/users/{id}Delete a user by ID
Gousers_api.go — complete CRUD service
package main

import (
    "encoding/json"
    "fmt"
    "net/http"
    "strconv"
    "sync"
)

type User struct {
    ID    int    `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

// In-memory store — protected by a RWMutex for concurrent access
var (
    users  = map[int]User{}
    nextID = 1
    mu     sync.RWMutex
)

// writeJSON is a helper that sets Content-Type and encodes the response
func writeJSON(w http.ResponseWriter, status int, v any) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    json.NewEncoder(w).Encode(v)
}

// GET /api/users
func listUsers(w http.ResponseWriter, r *http.Request) {
    mu.RLock()
    defer mu.RUnlock()

    list := make([]User, 0, len(users))
    for _, u := range users {
        list = append(list, u)
    }
    writeJSON(w, http.StatusOK, list)
}

// POST /api/users
func createUser(w http.ResponseWriter, r *http.Request) {
    var input User
    if err := json.NewDecoder(r.Body).Decode(&input); err != nil {
        writeJSON(w, http.StatusBadRequest, map[string]string{
            "error": "invalid JSON body",
        })
        return
    }
    if input.Name == "" || input.Email == "" {
        writeJSON(w, http.StatusBadRequest, map[string]string{
            "error": "name and email are required",
        })
        return
    }

    mu.Lock()
    input.ID = nextID
    nextID++
    users[input.ID] = input
    mu.Unlock()

    writeJSON(w, http.StatusCreated, input)
}

// GET /api/users/{id}
func getUser(w http.ResponseWriter, r *http.Request) {
    id, err := strconv.Atoi(r.PathValue("id"))
    if err != nil {
        writeJSON(w, http.StatusBadRequest, map[string]string{"error": "invalid id"})
        return
    }

    mu.RLock()
    u, ok := users[id]
    mu.RUnlock()

    if !ok {
        writeJSON(w, http.StatusNotFound, map[string]string{"error": "user not found"})
        return
    }
    writeJSON(w, http.StatusOK, u)
}

// DELETE /api/users/{id}
func deleteUser(w http.ResponseWriter, r *http.Request) {
    id, err := strconv.Atoi(r.PathValue("id"))
    if err != nil {
        writeJSON(w, http.StatusBadRequest, map[string]string{"error": "invalid id"})
        return
    }

    mu.Lock()
    _, ok := users[id]
    if ok { delete(users, id) }
    mu.Unlock()

    if !ok {
        writeJSON(w, http.StatusNotFound, map[string]string{"error": "user not found"})
        return
    }
    writeJSON(w, http.StatusNoContent, nil)
}

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /api/users",         listUsers)
    mux.HandleFunc("POST /api/users",        createUser)
    mux.HandleFunc("GET /api/users/{id}",    getUser)
    mux.HandleFunc("DELETE /api/users/{id}", deleteUser)

    fmt.Println("🚀 Users API running on http://localhost:8080")
    http.ListenAndServe(":8080", mux)
}

§6 Middleware — Cross-Cutting Concerns

Middleware wraps a handler with extra behaviour — logging, auth, CORS — without cluttering the handler itself. In Go, middleware is just a function that takes an http.Handler and returns a new http.Handler:

Gomiddleware — the Go pattern
// Middleware signature: takes a Handler, returns a Handler
func loggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)   // call the actual handler
        fmt.Printf("%s %s — %dms\n",
            r.Method, r.URL.Path, time.Since(start).Milliseconds())
    })
}

func authMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        token := r.Header.Get("Authorization")
        if token != "Bearer forge-secret" {
            writeJSON(w, http.StatusUnauthorized, map[string]string{
                "error": "unauthorized",
            })
            return   // stop — don't call next
        }
        next.ServeHTTP(w, r)
    })
}

// Chain middleware: outermost runs first
func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /api/users", listUsers)

    // Wrap mux: request flows logging → auth → mux → handler
    handler := loggingMiddleware(authMiddleware(mux))
    http.ListenAndServe(":8080", handler)
}

§7 HTTP Status Codes — Use Them Right

❌ lazy status codes
// Everything is 200 OK, even errors
w.WriteHeader(200)
json.NewEncoder(w).Encode(map[string]string{
    "error": "not found",
})
// client has to parse body to know
// if it succeeded. Terrible API.
✅ semantic status codes
// Use the right code every time
// 200 OK         — success GET/PUT
// 201 Created    — success POST
// 204 No Content — success DELETE
// 400 Bad Request — bad input
// 401 Unauthorized — no/bad auth
// 404 Not Found  — missing resource
// 500 Internal   — your bug
💡
Use http.Status* constants, not raw numbers

http.StatusOK, http.StatusCreated, http.StatusNotFound — Go's net/http package defines constants for every standard status code. Use them instead of magic numbers. They're self-documenting and spell-checked by the compiler.
